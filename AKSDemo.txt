--STEP2 — Create Azure Container Registry (ACR)

--Open Powershell as Administrator
--Azure Login

az login

Select subscription

--Validate if the correct subscription has picked

az account show

--Create Resource Group

az group create --name rg-aks-demo --location centralindia

--Create ACR (ACR name must be globally unique)

az acr create --resource-group rg-aks-demo --name myacraksdemo12345 --sku Basic

--Enable admin (for simplicity in POC) [Optional]

az acr update --name myacraksdemo12345 --admin-enabled true

--Get login server

az acr show --name myacraksdemo12345 --query loginServer --output table

Result
----------------------------
myacraksdemo12345.azurecr.io

STEP 3-Enable ACR Build from GitHub
------------------------------------------
We will tell Azure: 👉 “Build Docker image directly from GitHub source”

--Create build for Product API

az acr build --registry myacraksdemo12345 --image product-api:v1 --file ProductAPI/Dockerfile https://github.com/saurabhukumar/aksdemo.git

az acr build --registry myacraksdemo12345 --image frontend-api:v1 --file FrontendAPI/Dockerfile https://github.com/saurabhukumar/aksdemo.git

STEP 4 — Verify images in ACR

az acr repository list --name myacraksdemo12345 --output table

STEP 5 — Create AKS Cluster

az aks create --resource-group rg-aks-demo --name aks-demo --node-count 2 --attach-acr myacraksdemo12345 --generate-ssh-keys

--Connect Kubectl

az aks get-credentials --resource-group rg-aks-demo --name aks-demo

Output: Merged "aks-demo" as current context in C:\Users\saurabh.u.kumar\.kube\config

The Service provides: 
Stable DNS name
Stable IP
Load balancing across Pods

Without a Service, Pods are ephemeral.
